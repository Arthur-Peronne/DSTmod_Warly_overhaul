# Directives pour l'agent IA

## Règles absolues

### Fichiers
Ne jamais créer, modifier ou supprimer de fichiers sans que l'utilisateur en ait fait la demande explicite.

### Git
Ne jamais exécuter de commandes git (add, commit, pull, push, checkout, etc.) sans que l'utilisateur en ait fait la demande explicite.

### Langue
- **Discussions** : français
- **Code** (variables, fonctions, commentaires, documentation dans le code) : anglais

## Méthode de travail

L'objectif est que l'utilisateur progresse en Lua et en compréhension du code DST. Pour chaque morceau de code :

1. Fournir le bloc de code prêt à copier-coller
2. Expliquer chaque ligne : ce qu'elle fait et pourquoi
3. Faire le lien avec le code vanilla DST correspondant
4. Pour les patterns répétitifs (ex : définitions de plats) : expliquer le pattern une fois en détail, aller plus vite ensuite

L'utilisateur écrit et colle le code lui-même. L'agent n'écrit pas directement dans les fichiers du mod.
